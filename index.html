Interdisciplinary Projects & Research

Showcasing 2-3 major projects demonstrating the integration of multiple disciplines.

Landmannalaugar Terrain Renderer

Overview & Objectives: Real-world terrain visualization of Landmannalaugar, Iceland using C++/OpenGL and real satellite imagery.

Methodologies: Used C++ with OpenGL for graphics, OpenWeatherMap API for live weather, and managed particle systems for dynamic rain/snow.

Outcomes & Innovations: A 3D application featuring a geologically accurate terrain with a 9930x7240 pixel satellite texture, multi-texture triplanar mapping, and a dynamic day/night cycle.

Critical Analysis: The main challenge was optimizing the high-resolution texture and particle system to run smoothly. This was solved by implementing texture streaming and GPU-based particle simulation.

Hostel Management System

Overview & Objectives: Developed a full-stack web application to manage hostel bookings, student records, and room allocation as part of my internship at ByteWeavers Solutions.

Methodologies: Used HTML, CSS, and JavaScript for the frontend. Built the backend with PHP and managed the database with MySQL. Followed agile development practices.

Outcomes & Innovations: A production-ready web portal that streamlined hostel operations, replacing a manual, paper-based system.

Critical Analysis: The main challenge was ensuring data security and handling concurrent bookings. This was solved by implementing user authentication, session management, and database-level transaction locks.

Weather Alert Notification System

Overview & Objectives: Developed a Python-based system that fetches real-time weather data from an API and alerts users about severe weather conditions.

Methodologies: Used Python's Tkinter library for the GUI, the OpenWeatherMap API for data, and SQLite for storing user preferences and thresholds.

Outcomes & Innovations: A functional desktop application where users can set custom thresholds (e.g., wind speed, temperature) and receive email alerts via SMTP.

Critical Analysis: Implementing a non-blocking API call was a challenge in Tkinter's main loop. This was solved using multi-threading to fetch data in the background.

Global Awareness & Ethical Considerations

Case Study: Bias in NLP Sentiment Analysis

While working on sentiment analysis models, I encountered a significant ethical dilemma: dataset bias. Models trained on public internet data often inherit societal biases, misclassifying text from certain demographics or contexts (e.g., treating AAVE as 'negative').

Analysis of Implications: The implication is that a biased model could unfairly penalize or misrepresent a user's intent, leading to discriminatory outcomes in applications like customer service bots or content moderation. This is a global issue as AI systems become more integrated into daily life.

Personal Reflection: This experience taught me that technical correctness isn't enough. As an engineer, I have a responsibility to actively audit my data, test for bias, and implement fairness-aware techniques. It reinforced my commitment to building technology that is not only smart but also equitable.
